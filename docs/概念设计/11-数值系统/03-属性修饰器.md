# 《开天辟地之大地的裂变》属性修饰器

## 1. 文档目的

属性修饰器（Attribute Modifier）用于描述角色属性在基础输入之外受到的额外修正。

本系统中的属性计算由以下几类数据共同完成：

* 当前基础属性
* 基础属性计算系数
* 基础属性静态偏移量
* 基础属性动态偏移量
* 派生属性静态偏移量
* 派生属性动态偏移量

其中，系数与偏移量虽然都会影响最终结果，但两者的职责并不相同：

* 系数决定基础属性如何转化为派生属性
* 偏移量决定某项属性在原有结果上增加或减少多少

本章节主要说明属性修饰器的类型、来源与生命周期，并明确游戏中的哪些内容属于偏移量，哪些内容不属于偏移量。

具体计算公式与结算顺序将在《04-属性计算与结算顺序》中统一说明。

---

## 2. 属性数据的整体组成

角色参与派生属性计算时，需要使用以下几类数据：

| 数据类型 | 作用 | 示例 |
| --- | --- | --- |
| 当前基础属性 | 作为派生属性计算的基础输入 | 力量 8、体质 6、灵力 4 |
| `coefficients` | 决定每项基础属性对当前派生属性的影响程度 | 体质每点提供一定最大生命值 |
| `primaryStaticOffset` | 在计算某项派生属性时，对基础属性进行固定修正 | 计算最大生命值时，将体质视为额外增加 2 点 |
| `primaryDynamicOffset` | 运行过程中临时修正角色基础属性 | Buff 使力量临时增加 2 点 |
| `derivedStaticOffset` | 对某项派生属性提供固定基础值 | 最大生命值固定增加 20 点 |
| `derivedDynamicOffset` | 运行过程中直接修正某项派生属性 | 装备提供护甲穿透 3 点 |

这些数据虽然最终都会参与属性计算，但来源、用途和生命周期不同，不应混为一谈。

原剧中的职业、装备、卡牌、天气、建筑和突发事件共同构成了《开天辟地之大地的裂变》规则繁多、效果频发的游戏气质。属性修饰器的作用不是削弱这种复杂感，而是将不同系统产生的数值变化收束为统一结构，使其能够被准确叠加、移除、追踪和解释。

---

## 3. 当前基础属性不是属性修饰器

角色最开始参与计算的基础输入为：

```text
当前基础属性
=
统一初始值
+
种族调整
+
职业调整
+
升级加点
+
转职成长
+
其他永久成长
```

本项目中，五项基础属性的统一初始值均为 5。

角色创建时，种族和职业可能重新分配这五项属性，但必须保证初始基础属性总量一致。

例如：

```text
统一初始属性：
力量 5、体质 5、灵力 5、敏捷 5、悟性 5
总量为 25
```

经过种族与职业调整后，角色可能变为：

```text
力量 7、体质 6、灵力 3、敏捷 5、悟性 4
总量仍为 25
```

在角色创建结束后，这些结果将被合并并保存为角色的当前基础属性。

因此，以下内容虽然会改变基础属性，但在最终数据结构中通常不作为属性修饰器保存：

* 基础属性统一初始值
* 种族带来的初始属性调整
* 职业带来的初始属性调整
* 升级时玩家分配的属性点
* 转职获得的永久属性成长
* 任务给予的永久属性点
* 剧情产生的永久成长
* 其他已经永久沉淀到角色身上的属性变化

这些内容共同决定角色当前拥有多少基础属性，是派生属性计算的原始输入，而不是每次计算时额外叠加的偏移量。

---

## 4. 为什么永久成长不作为动态偏移量

升级加点、转职成长和永久任务奖励虽然都发生在游戏过程中，但它们不会随时间消失，也不需要在效果结束时移除。

例如，角色升级时将一点属性分配给力量：

```text
升级前力量：7
升级加点：+1
升级后力量：8
```

系统应当直接将角色的当前力量更新为 8，而不是长期保留一条“升级力量 +1”的动态修饰器。

这样设计具有以下优点：

* 角色当前基础属性可以直接读取
* 不需要永久保存大量历史修饰器
* 加载角色时无需重新回放所有成长记录
* 派生属性计算过程更加简单
* 临时效果与永久成长之间界限清晰

因此，判断某项内容是否属于动态偏移量，关键不在于它是否发生在游戏过程中，而在于：

**该效果是否具有可变性，并且在特定条件下需要被移除。**

## 5. 系数

### 5.1 系数的作用

`coefficients` 用于定义基础属性与派生属性之间的换算关系。

例如，最大生命值仅受到体质影响：

```json
{
  "coefficients": {
    "strength": 0,
    "constitution": 8,
    "spirit": 0,
    "agility": 0,
    "insight": 0
  }
}
```

这表示只有体质参与最大生命值计算，其余基础属性不影响最大生命值。

### 5.2 系数不是偏移量

系数不属于属性偏移量。

两者区别如下：

| 类型 | 作用 |
| --- | --- |
| 系数 | 决定基础属性按照什么比例转换为派生属性 |
| 偏移量 | 在原有属性或计算结果上直接增加、减少数值 |

例如：

```text
力量系数为 2
```

表示每一点力量按照一定比例参与派生属性计算。

而：

```text
物理攻击 +2
```

表示直接为最终物理攻击增加 2 点。

虽然两者最终都可能改变物理攻击，但其计算位置和设计意义完全不同。

### 5.3 系数的来源

系数一般由作者在派生属性配置中预先定义。

系数属于数值系统固定规则，通常不会在游戏运行过程中变化。

例如：

* 最大生命值只受到体质影响
* 移动速度只受到敏捷影响
* 法术攻击受到灵力和悟性影响
* 护甲穿透不受任何基础属性影响，其所有系数均为零

如果某个技能需要临时改变基础属性对派生属性的换算关系，应将其视为少量特殊机制，不能直接混入普通偏移量体系。

---

## 6. 偏移量分类

属性偏移量按照作用对象分为：

* 基础属性偏移量
* 派生属性偏移量

按照生命周期分为：

* 静态偏移量
* 动态偏移量

最终形成四种偏移量：

| 偏移量 | 作用层级 | 生命周期 |
| --- | --- | --- |
| `primaryStaticOffset` | 基础属性层 | 固定 |
| `primaryDynamicOffset` | 基础属性层 | 运行时 |
| `derivedStaticOffset` | 派生属性层 | 固定 |
| `derivedDynamicOffset` | 派生属性层 | 运行时 |

---

## 7. 基础属性静态偏移量

### 7.1 定义

`primaryStaticOffset` 是作者在某项派生属性配置中预先设置的基础属性固定修正。

它不会修改角色实际拥有的基础属性，只在计算当前派生属性时使用。

例如：

```json
{
  "maxHealth": {
    "primaryStaticOffset": {
      "strength": 0,
      "constitution": 2,
      "spirit": 0,
      "agility": 0,
      "insight": 0
    }
  }
}
```

该配置表示：

在计算最大生命值时，角色的体质按照“当前体质额外增加 2 点”参与计算。

这不会让角色面板上的体质真正增加，也不会影响物理防御、闪避或其他派生属性。

### 7.2 适用场景

基础属性静态偏移量适合表达作者定义的固定计算基准，例如：

* 某项派生属性拥有隐藏的基础属性起点
* 为低属性角色提供固定的计算底数
* 调整某项派生属性的整体成长曲线
* 在不修改角色基础属性的情况下，对单项派生属性进行固定补偿
* 为某些特殊派生属性建立统一起始标准

### 7.3 不适用场景

以下内容不应放入 `primaryStaticOffset`：

* 种族初始属性
* 职业初始属性
* 升级加点
* 装备提供的基础属性
* Buff 提供的基础属性
* 天气产生的临时属性变化
* 角色永久成长

种族、职业和升级成长应当计入当前基础属性；装备、Buff 和天气等运行时效果应当计入 `primaryDynamicOffset`。

---

## 8. 基础属性动态偏移量

### 8.1 定义

`primaryDynamicOffset` 是游戏运行过程中对基础属性产生的可变修正。

它会影响所有依赖对应基础属性的派生属性。

例如：

```text
力量临时 +2
```

可能同时提高物理攻击、物理防御、暴击伤害、负重或其他以力量为输入的派生属性。

因此，基础属性动态偏移量属于影响范围较广的修饰器。

### 8.2 典型来源

以下内容通常属于 `primaryDynamicOffset`：

| 来源 | 示例 |
| --- | --- |
| 装备 | 武器使力量 +2 |
| Buff | 激励状态使悟性 +1 |
| Debuff | 虚弱状态使体质 -2 |
| 药剂 | 一定回合内灵力 +3 |
| 天气 | 台风天气使敏捷 -1 |
| 地图环境 | 沼泽区域使敏捷 -2 |
| 建筑效果 | 神殿范围内灵力 +1 |
| 光环 | 队友光环使体质 +1 |
| 卡牌效果 | 本回合力量 +3 |
| 临时形态 | 变身期间力量 +4、悟性 -2 |
| 临时事件 | 受到惊吓后悟性 -1，持续两回合 |

### 8.3 作用特点

基础属性动态偏移量具有以下特点：

* 会影响多个派生属性
* 具有明确来源
* 可能具有持续时间
* 在来源失效后需要移除
* 可以是正数，也可以是负数
* 不修改角色永久基础属性

### 8.4 使用限制

由于基础属性会同时影响多个派生属性，因此不应轻易使用较大的基础属性动态偏移量。

例如，一个装备直接提供力量 +5，可能同时提高：

* 物理攻击
* 物理防御
* 暴击伤害
* 负重
* 采集效率
* 其他力量关联属性

如果作者只想让装备提升物理攻击，就不应增加力量，而应使用 `derivedDynamicOffset` 直接增加物理攻击。

---

## 9. 派生属性静态偏移量

### 9.1 定义

`derivedStaticOffset` 是作者在派生属性配置中写入的固定基础值。

它不依赖基础属性，直接加入对应派生属性的计算结果。

例如：

```json
{
  "maxHealth": {
    "derivedStaticOffset": 20
  }
}
```

表示所有角色在基础属性换算结果之外，都会获得固定的 20 点最大生命值。

### 9.2 典型用途

派生属性静态偏移量适合用于：

* 为派生属性设置固定基础值
* 保证低基础属性角色仍具有基本游戏能力
* 调整某项派生属性的整体数值区间
* 为不受基础属性影响的派生属性设置初始值
* 避免通过修改系数改变属性成长斜率

例如：

* 所有角色拥有固定基础生命值
* 所有角色拥有固定基础移动力
* 某项抗性默认从零开始
* 某项行动能力拥有统一基础值

### 9.3 与基础属性静态偏移量的区别

两者虽然都是作者预设的固定内容，但作用阶段不同。

| 类型 | 作用方式 |
| --- | --- |
| `primaryStaticOffset` | 先修正基础属性，再乘以系数 |
| `derivedStaticOffset` | 在派生属性换算结束后直接增加结果 |

因此，它们不能互相替代。

例如，体质静态增加 2 点会随着体质系数放大；最大生命值静态增加 20 点则永远只增加 20 点。

---

## 10. 派生属性动态偏移量

### 10.1 定义

`derivedDynamicOffset` 是游戏运行过程中直接作用于某项派生属性的临时或可变修正。

它只影响指定的派生属性，不会扩散到其他属性。

例如：

```text
护甲穿透 +3
```

只提高护甲穿透，不会增加物理攻击、力量或其他能力。

### 10.2 典型来源

以下内容通常属于 `derivedDynamicOffset`：

| 来源 | 示例 |
| --- | --- |
| 装备基础数值 | 武器提供物理攻击 +4 |
| 装备词条 | 护甲穿透 +3 |
| Buff | 本回合移动速度 +2 |
| Debuff | 受到治疗效果 -2 |
| 技能效果 | 三回合内法术抗性 +5 |
| 药剂 | 临时最大生命值 +10 |
| 地图效果 | 高地提供视野 +1 |
| 天气 | 黑夜使视野 -1 |
| 建筑 | 铁匠铺中制造能力 +2 |
| 卡牌 | 下一次鉴定能力 +3 |
| 事件 | 幸运值临时 +1 |
| 光环 | 周围友军物理防御 +2 |

### 10.3 适用场景

当某个效果只需要修改一项明确能力时，应优先使用派生属性动态偏移量。

例如：

* 武器需要增加物理攻击
* 破甲装备需要增加护甲穿透
* 靴子需要增加移动力
* 特殊道具需要增加幸运值
* 鉴定工具需要增加鉴定能力
* 护目镜需要增加感知或视野

这种设计可以防止装备因修改基础属性而意外提高大量无关能力。

## 11. 哪些内容算作偏移量

### 11.1 属于偏移量的内容

以下内容通常属于属性偏移量：

* 装备提供的属性加减
* 装备词条提供的特殊数值
* Buff 与 Debuff
* 药剂的临时效果
* 天气对角色能力的影响
* 地图地形与特殊地点效果
* 建筑效果
* 技能的持续属性修正
* 光环效果
* 卡牌带来的临时增减
* 事件产生的限时影响
* 临时变身与状态切换
* 作者为派生属性设置的固定补偿值
* 作者为单项派生属性设置的固定基础属性修正

这些内容的共同特点是：

它们是在角色当前基础输入或派生属性结果之外产生的额外增减。

### 11.2 不属于偏移量的内容

以下内容不属于属性偏移量：

| 内容 | 原因 |
| --- | --- |
| 五项属性的统一初始值 | 属于角色基础输入 |
| 种族初始属性调整 | 创建完成后沉淀为当前基础属性 |
| 职业初始属性调整 | 创建完成后沉淀为当前基础属性 |
| 升级属性加点 | 属于永久成长，直接更新当前基础属性 |
| 转职永久成长 | 属于永久成长，直接更新当前基础属性 |
| 永久任务奖励 | 直接更新当前基础属性 |
| 派生属性计算系数 | 属于换算规则，不是数值增减 |
| 当前生命值 | 属于资源状态，不是属性修饰器 |
| 当前灵力值 | 属于资源状态，不是属性修饰器 |
| 受到一次伤害 | 属于结算事件，不是属性偏移量 |
| 消耗一次灵力 | 属于资源扣除，不是属性偏移量 |
| 移动消耗 | 属于行动结算，不是属性偏移量 |
| 经验值 | 属于成长资源，不是属性偏移量 |
| 元宝 | 属于经济资源，不是属性偏移量 |
| 状态名称本身 | 状态可以携带偏移量，但状态本身不是数值 |
| 技能伤害倍率 | 属于技能结算规则，不是属性偏移量 |
| 地形移动消耗 | 属于地图规则，不是角色属性偏移量 |

## 12. 静态偏移量与动态偏移量的判断方式

### 12.1 静态偏移量

满足以下特征时，通常属于静态偏移量：

* 由作者在派生属性配置中预先定义
* 所有符合条件的角色始终使用相同规则
* 游戏运行过程中通常不会改变
* 不需要记录来源、持续时间或剩余回合
* 主要用于构建属性基础值与调整整体计算曲线

典型内容包括：

* 最大生命值的固定基础值
* 移动力的固定起始值
* 某项派生属性计算时使用的基础属性固定修正
* 不受基础属性影响的派生属性默认值

### 12.2 动态偏移量

满足以下特征时，通常属于动态偏移量：

* 在游戏运行过程中产生
* 不同角色可能拥有不同数值
* 具有装备、技能、状态、地图或事件等具体来源
* 可能被添加、移除、替换或刷新
* 可能具有持续时间或触发条件
* 需要随着游戏状态重新计算

典型内容包括：

* 装备提供的属性
* Buff 与 Debuff
* 天气修正
* 地图区域修正
* 建筑范围效果
* 光环
* 卡牌效果
* 临时任务效果
* 事件产生的限时增减

---

## 13. 装备属性的处理原则

装备既可以提供基础属性动态偏移量，也可以提供派生属性动态偏移量。

例如：

```text
巨力护腕：
力量 +1
```

属于：

```text
primaryDynamicOffset.strength += 1
```

由于力量可能影响多个派生属性，该装备会产生综合成长效果。

又例如：

```text
破甲长枪：
物理攻击 +3
护甲穿透 +2
```

属于：

```text
derivedDynamicOffset.physicalAttack += 3
derivedDynamicOffset.armorPenetration += 2
```

该装备只会提高指定能力，不会增加角色力量。

装备设计时应根据实际意图选择偏移层级：

* 希望装备带来综合成长时，使用基础属性动态偏移量
* 希望装备强化明确功能时，使用派生属性动态偏移量
* 穿透、幸运值、固定减伤等独立属性，原则上使用派生属性动态偏移量

## 14. 状态与偏移量的关系

Buff、Debuff、天气和地图效果本身不是偏移量，而是偏移量的来源。

例如：

```text
状态：虚弱
持续时间：2 回合
效果：力量 -2，移动力 -1
```

该状态会产生两个偏移量：

```text
primaryDynamicOffset.strength -= 2
derivedDynamicOffset.movementRange -= 1
```

状态系统负责管理：

* 来源
* 持续时间
* 层数
* 刷新规则
* 移除条件

属性系统只负责汇总该状态当前提供的数值。

因此，状态系统与属性系统应保持职责分离。

---

## 15. 动态偏移量的管理方式

运行时不建议只保存一个无法追踪来源的最终合计值。

推荐为每个动态修饰器记录来源信息，例如：

```json
{
  "sourceId": "equipment_iron_boots",
  "sourceType": "equipment",
  "targetType": "derived",
  "targetAttribute": "movementRange",
  "value": 1,
  "duration": null
}
```

又例如：

```json
{
  "sourceId": "buff_weakness",
  "sourceType": "buff",
  "targetType": "primary",
  "targetAttribute": "strength",
  "value": -2,
  "duration": 2
}
```

系统可以在计算前，将所有有效修饰器汇总为：

* `primaryDynamicOffset`
* `derivedDynamicOffset`

这样既能保证计算效率，也能追踪每项偏移量的来源。

---

## 16. 动态偏移量的叠加原则

动态偏移量的具体叠加方式将在后续状态与装备系统中进一步定义，但数值系统应至少支持以下行为：

* 同来源刷新
* 不同来源叠加
* 限制最大层数
* 替换较低数值
* 取最高值
* 取最低值
* 效果到期移除
* 装备卸下后移除
* 离开区域后移除

属性计算模块不负责判断某个效果是否可以叠加，而只负责读取最终有效的动态偏移量。

## 17. 完整示例

假设某角色当前基础属性为：

```json
{
  "strength": 8,
  "constitution": 6,
  "spirit": 4,
  "agility": 7,
  "insight": 5
}
```

其中，这些数值已经包含：

* 初始属性
* 种族调整
* 职业调整
* 升级加点
* 永久成长

角色当前装备和状态如下：

```text
巨力护腕：力量 +1
疾行靴：移动力 +2
破甲长枪：护甲穿透 +3
虚弱状态：体质 -1
高地效果：视野 +1
```

系统汇总后得到：

```json
{
  "primaryDynamicOffset": {
    "strength": 1,
    "constitution": -1,
    "spirit": 0,
    "agility": 0,
    "insight": 0
  },
  "derivedDynamicOffset": {
    "movementRange": 2,
    "armorPenetration": 3,
    "visionRange": 1
  }
}
```

这里：

* 巨力护腕和虚弱状态影响基础属性
* 疾行靴、破甲长枪和高地效果直接影响派生属性
* 种族、职业和升级加点已经进入当前基础属性，不再作为修饰器重复计算

## 18. 设计原则

属性修饰器系统遵循以下原则：

### 18.1 输入明确原则

初始值、种族、职业、升级加点和永久成长共同构成当前基础属性，不作为运行时修饰器重复保存。

### 18.2 系数独立原则

系数用于定义基础属性到派生属性的换算关系，不属于偏移量。

### 18.3 静态配置原则

静态偏移量由作者在派生属性配置中定义，游戏运行过程中通常不会改变。

### 18.4 动态来源原则

动态偏移量必须具有明确来源，例如装备、Buff、天气、地图或事件。

### 18.5 永久沉淀原则

不会被移除的永久成长，应直接写入当前基础属性，而不是长期保留为动态偏移量。

### 18.6 影响范围原则

基础属性偏移量会影响多个派生属性，应谨慎使用；派生属性偏移量只影响指定能力，更适合功能明确的装备和效果。

### 18.7 职责分离原则

状态、装备和地图系统负责管理修饰器来源与生命周期，属性系统只负责汇总和计算。

### 18.8 可追踪原则

动态修饰器应尽量保留来源信息，方便移除、刷新、展示和问题排查。
