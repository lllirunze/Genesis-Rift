# 《开天辟地之大地的裂变》派生属性

## 1. 文档目的

派生属性（Derived Attributes）是角色实际参与战斗、移动、探索、制造、交易和随机事件等游戏机制的能力值。

与基础属性不同，派生属性通常不作为角色永久成长数据直接保存，而是根据以下内容实时计算：

* 角色当前基础属性
* 派生属性固定配置
* 基础属性动态偏移量
* 派生属性动态偏移量

本章节主要介绍派生属性的设计原则、初步分类和建议属性，不定义具体计算系数。详细计算方式将在《04-属性计算与结算顺序》中说明。

---

## 2. 派生属性的设计意义

五项基础属性描述角色的发展方向，而派生属性负责将这些发展方向转化为具体的游戏能力。

同一项基础属性可以影响多项派生属性，但不能让同一类能力全部沿着完全相同的路线自然成长。

例如，如果力量同时显著提高：

* 物理攻击
* 物理防御
* 护甲穿透
* 暴击伤害
* 负重能力

那么力量将成为一个几乎没有取舍的通用属性。角色只需要不断增加力量，便能同时获得输出、生存和突破防御的能力，进而压缩其他属性的发展空间。

因此，派生属性之间必须具有差异化，使角色在成长过程中面对真实的选择：

提高一种能力时，不应自然获得所有相关能力。

角色可以拥有较高的物理攻击，但未必拥有较高的护甲穿透；也可以拥有较高的行动速度，但未必拥有较远的移动距离。

这种差异是角色构筑、职业分化、装备选择和游戏可玩性的重要来源。

同时，本项目需要尊重原剧中“系统很多、职业很多、能力很多”的设计气质，但不能把这种复杂直接写成失控的数值膨胀。派生属性系统的作用，就是在保留展博式夸张感的前提下，为这些能力建立统一、可运行、可维护的换算层。

---

## 3. 基础属性参与数量

每项派生属性可以受到零至三项基础属性影响。

一般情况下，建议遵循以下规则：

| 影响数量 | 适用情况 |
| --- | --- |
| 0 项 | 适合穿透、固定减伤、幸运值等需要独立成长或主要由装备、事件提供的属性。 |
| 1 项 | 适合方向明确、功能单一的能力。 |
| 2 项 | 作为大部分派生属性的主要设计方式，用于形成不同属性组合。 |
| 3 项 | 仅用于少量综合能力，避免角色因属性覆盖过广而缺乏成长差异。 |

未参与某项派生属性计算的基础属性，其对应系数设置为 `0`。

基础属性参与数量并不代表各属性影响程度相同。即使同一项派生属性受到两项基础属性影响，也可以设置主属性与次属性，使其形成明确的发展倾向。

其中，最大生命值、最大灵力值、负重和移动速度属于功能边界明确的能力，分别仅由体质、灵力、力量和敏捷决定，用于体现单一基础属性对其核心领域的直接影响。

## 4. 派生属性差异化原则

### 4.1 同类能力不采用完全相同的属性组合

功能相似的派生属性应尽量采用不同的基础属性组合。

例如：

* 物理攻击可以由力量和敏捷共同影响
* 暴击率可以由敏捷和悟性共同影响
* 护甲穿透不受基础属性影响，只通过装备和特殊效果获得

这样可以避免物理输出、暴击和穿透随着同一项基础属性同步增长。

### 4.2 攻击与穿透分离

攻击属性代表角色造成伤害的基础能力。

穿透属性代表角色突破目标防御体系的能力。

两者虽然共同参与伤害结算，但属于不同的发展方向。因此，穿透属性原则上不受基础属性影响，而主要通过以下方式获得：

* 武器和装备
* 装备词条
* 特殊技能
* 临时 Buff
* 药剂
* 事件效果
* 职业特性
* 稀有道具

角色即使拥有很高的攻击力，也不应自动获得穿透能力。

### 4.3 移动力与移动速度分离

移动力与移动速度是两个不同概念。

移动力决定角色在自己的移动阶段内能够移动多少距离，主要参与地图移动和地形消耗。

移动速度决定同一回合中多个角色的移动先后顺序，主要参与行动排序。

移动速度仅由敏捷决定，不受悟性或其他基础属性影响。

因此，一个角色可以：

* 移动距离较远，但行动顺序较慢
* 行动顺序较快，但单次移动距离较短
* 两项能力都较高，但需要为此投入更多成长资源

### 4.4 随机能力独立发展

幸运值用于影响后续可能加入的随机系统，但不直接决定某次随机结果。

幸运值可以参与：

* 随机事件结果修正
* 掉落结果修正
* 制造意外结果
* 抽卡结果微调
* 命运卡或事件卡判定
* 同等条件下的随机优先级
* 部分高风险行为的结果倾向

幸运值原则上不受基础属性影响，主要通过角色特性、装备、事件和临时效果获得。

这样可以避免悟性或敏捷在获得原本能力的同时，自然取得随机系统优势。

## 5. 建议派生属性

以下派生属性为当前阶段的概念规划。表格只描述建议的基础属性关系和主要用途，具体系数、静态偏移量及边界规则以后续配置为准。

| 派生属性 | 英文名 | 建议影响基础属性 | 主要参与部分 |
| --- | --- | --- | --- |
| 最大生命值 | Max Health | 体质 | 战斗、生存、伤害承受 |
| 最大灵力值 | Max Mana | 灵力 | 技能释放、法术消耗、特殊行动 |
| 生命恢复 | Health Regeneration | 体质、悟性 | 回合恢复、休整、持续战斗 |
| 灵力恢复 | Mana Regeneration | 灵力、体质 | 技能循环、休整、持续施法 |
| 物理攻击 | Physical Attack | 力量、敏捷 | 普通攻击、物理技能、武器伤害 |
| 法术攻击 | Magic Attack | 灵力、悟性 | 法术技能、精神攻击、特殊能力 |
| 物理防御 | Physical Defense | 体质、力量 | 抵御物理伤害、环境冲击 |
| 法术抗性 | Magic Resistance | 体质、灵力 | 抵御法术伤害、精神效果 |
| 命中 | Accuracy | 敏捷、悟性 | 攻击命中、远程攻击、精确操作 |
| 闪避 | Evasion | 敏捷、体质 | 躲避攻击、脱离危险、陷阱规避 |
| 暴击率 | Critical Chance | 敏捷、悟性 | 战斗中的暴击判定 |
| 暴击伤害 | Critical Damage | 力量、灵力 | 物理或法术暴击后的伤害放大 |
| 护甲穿透 | Armor Penetration | 无 | 突破物理防御，主要由装备和效果提供 |
| 法术穿透 | Magic Penetration | 无 | 突破法术抗性，主要由装备和效果提供 |
| 固定伤害减免 | Flat Damage Reduction | 无 | 直接削减受到的伤害，主要由护具和特殊状态提供 |
| 治疗强度 | Healing Power | 灵力、体质 | 治疗技能、药剂增幅、辅助能力 |
| 受治疗效果 | Healing Received | 体质、悟性 | 接受治疗、休整、恢复类道具 |
| 移动力 | Movement Range | 体质、敏捷 | 单次移动距离、地形移动、追击与撤退 |
| 移动速度 | Movement Speed | 敏捷 | 同一回合中的移动先后顺序 |
| 行动速度 | Action Speed | 敏捷、灵力 | 战斗行动顺序、技能释放顺序 |
| 视野 | Vision Range | 悟性、灵力 | 地图可见范围、侦察、发现目标 |
| 感知 | Perception | 悟性、敏捷 | 发现陷阱、隐藏角色、秘密地点和特殊事件 |
| 隐匿 | Stealth | 敏捷、灵力 | 潜行、隐藏信息、规避侦察 |
| 意志 | Willpower | 悟性、体质 | 抵抗控制、恐惧、诱惑和精神干扰 |
| 控制抗性 | Control Resistance | 体质、灵力 | 减少眩晕、减速和限制效果 |
| 负重 | Carry Capacity | 力量 | 携带装备、材料和任务物品 |
| 采集效率 | Gathering Efficiency | 力量、悟性 | 资源采集、材料获取、地图互动 |
| 制造能力 | Crafting Ability | 悟性、敏捷 | 装备打造、道具制作、材料处理 |
| 鉴定能力 | Identification Ability | 悟性、灵力 | 未知装备、未知图纸和特殊物品鉴定 |
| 交涉能力 | Negotiation | 悟性、灵力 | NPC 交流、情报购买、任务交涉 |
| 商业能力 | Commerce | 悟性、敏捷 | 商店价格、交易收益、特殊商品发现 |
| 幸运值 | Luck | 无 | 随机系统、事件结果、掉落和命运判定 |
| 裂变抗性 | Rift Resistance | 体质、灵力、悟性 | 地图裂变、异常空间和特殊灾害 |
| 裂变感知 | Rift Perception | 悟性、灵力 | 预判地图裂变、发现异常区域和裂变线索 |
| 逃脱能力 | Escape Ability | 敏捷、体质 | 脱离战斗、逃离危险区域、摆脱追踪 |

其中需要特别强调的是，诸如裂变抗性、裂变感知、任务洞察等属性虽然是为了补完数字桌游化后的完整系统而新增，但它们仍然服务于原剧已经存在的创界山、世界裂变、隐藏信息和多目标博弈气质，而不是将本作改写成另一套完全无关的数值框架。

## 6. 相似属性的差异化说明

为了避免角色沿单一属性路线获得过多能力，以下相似属性必须保持明确差异。

| 属性组合 | 差异说明 |
| --- | --- |
| 物理攻击与护甲穿透 | 物理攻击由基础属性成长；护甲穿透不受基础属性影响，主要依靠装备与特殊效果。 |
| 法术攻击与法术穿透 | 法术攻击由灵力、悟性成长；法术穿透作为独立装备和技能路线。 |
| 物理防御与固定伤害减免 | 物理防御由体质、力量成长；固定减伤不受基础属性影响，主要来自护具和特殊状态。 |
| 命中与暴击率 | 命中保证攻击能够生效；暴击率决定攻击生效后的额外收益，两者基础属性组合可以相似但系数倾向应不同。 |
| 闪避与移动速度 | 闪避用于避免攻击；移动速度用于决定移动顺序，不能将两者视为同一能力。 |
| 移动力与移动速度 | 移动力决定走多远；移动速度决定谁先移动。 |
| 行动速度与先手值 | 行动速度可持续影响回合内行动顺序；先手值主要用于战斗或事件开始时的第一次排序。 |
| 视野与感知 | 视野决定能够看到多远；感知决定能否发现隐藏内容。 |
| 制造能力与鉴定能力 | 制造能力负责生产；鉴定能力负责揭露未知信息。 |
| 交涉能力与商业能力 | 交涉能力影响对话与任务；商业能力影响交易和商品。 |
| 幸运值与悟性 | 悟性代表理解与判断；幸运值代表随机倾向，两者不直接绑定。 |
| 裂变抗性与裂变感知 | 裂变抗性用于承受裂变影响；裂变感知用于提前发现和判断裂变。 |

## 7. 无基础属性影响的派生属性

部分派生属性的五项基础属性系数可以全部为 `0`。

这类属性不通过基础属性自然成长，而主要依赖自身的静态偏移量和动态偏移量。

典型属性包括：

* 护甲穿透
* 法术穿透
* 固定伤害减免
* 幸运值
* 某些特殊元素穿透
* 特定阵营抗性
* 特殊事件修正值

其概念结构如下：

```json
{
  "armorPenetration": {
    "coefficients": {
      "strength": 0,
      "constitution": 0,
      "spirit": 0,
      "agility": 0,
      "insight": 0
    },
    "primaryStaticOffset": {
      "strength": 0,
      "constitution": 0,
      "spirit": 0,
      "agility": 0,
      "insight": 0
    },
    "derivedStaticOffset": 0
  }
}
```

角色最终拥有的护甲穿透主要来自 `derivedDynamicOffset`，例如：

* 武器提供护甲穿透 `+3`
* 装备词条提供护甲穿透 `+2`
* 临时 Buff 提供护甲穿透 `+1`

这些来源可以共同构成角色当前的护甲穿透能力，但不会永久修改任何基础属性。

## 8. 派生属性配置结构

每项派生属性拥有独立配置，主要包含：

| 配置项 | 说明 |
| --- | --- |
| `coefficients` | 五项基础属性对应的系数，没有影响时设置为 `0`。 |
| `primaryStaticOffset` | 仅在计算当前派生属性时使用的基础属性静态偏移量。 |
| `derivedStaticOffset` | 直接作用于该派生属性结果的静态偏移量。 |

示例：

```json
{
  "maxHealth": {
    "coefficients": {
      "strength": 0,
      "constitution": 8,
      "spirit": 0,
      "agility": 0,
      "insight": 0
    },
    "primaryStaticOffset": {
      "strength": 0,
      "constitution": 0,
      "spirit": 0,
      "agility": 0,
      "insight": 0
    },
    "derivedStaticOffset": 20
  }
}
```

该配置只说明最大生命值仅受到体质影响，不表示表格中的建议系数已经最终确定。

具体平衡数值将在数值测试阶段持续调整。

## 9. 角色发展差异

派生属性的设计目的不是让所有角色最终获得相同能力，而是让玩家通过不同选择形成不同的发展路线。

例如，同样偏向物理战斗的角色，可以分别发展为：

* 高攻击但低穿透的正面输出者
* 攻击一般但依靠装备获得高穿透的破甲者
* 高暴击率但单次伤害较低的快速攻击者
* 高移动速度、擅长抢占位置的先行者
* 高移动力、擅长追击与撤退的游击角色
* 高命中、擅长处理高闪避目标的稳定输出者

同样偏向悟性的角色，也可以分别发展为：

* 法术输出者
* 制造者
* 鉴定者
* 情报与任务探索者
* 裂变感知者
* NPC 交涉者

因此，基础属性不是简单对应固定职业，而是通过不同的派生属性组合形成多种角色路线。

## 10. 设计原则

派生属性系统遵循以下原则：

### 10.1 差异化原则

功能相似的派生属性不能完全依赖同一组基础属性，避免同类能力同步增长。

### 10.2 有限关联原则

每项派生属性最多受到三项基础属性影响，大部分属性建议由两项基础属性共同决定。

### 10.3 独立成长原则

穿透、固定减伤、幸运值等特殊能力不受基础属性影响，通过装备、Buff、事件和其他独立来源成长。

### 10.4 主次明确原则

受到多项基础属性影响时，应区分主要属性与次要属性，避免所有系数完全相同。

### 10.5 玩法解耦原则

派生属性只提供数值，不负责具体玩法逻辑；战斗、移动、探索和制造等系统使用这些数值完成各自判定。

### 10.6 配置驱动原则

派生属性的基础属性组合、系数和静态偏移量均由资源文件维护，不写入程序逻辑。

### 10.7 可玩性原则

派生属性之间应形成选择与取舍，使不同角色即使拥有相近的基础属性，也能够通过装备、职业、转职和事件形成不同的发展路线。
