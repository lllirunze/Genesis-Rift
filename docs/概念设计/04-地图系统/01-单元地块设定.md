# 《开天辟地之大地的裂变》单元地块设定

## 1. 文档目的

本文定义《开天辟地之大地的裂变》中地图的最小组成单位：单元地块（Grid Tile）。

单元地块仅负责描述地图空间，不承担角色、事件、任务、战斗等游戏逻辑。

所有复杂玩法均通过其他系统附加到地块上，而不是写入地块本身。

---

## 2. 设计原则

单元地块遵循以下原则。

### 2.1 最小空间单位

单元地块是整个世界地图中最小且不可再拆分的空间单位。

玩家移动、距离计算、攻击范围、寻路等均以单元地块作为基础。

在首版地图中，所有单元地块共同组成一个以中心点为 `0` 环、向外扩散至 `10` 环、`21 × 21` 的正方形网格地图。

### 2.2 单一职责

单元地块只负责：

* 描述空间
* 维护相邻关系
* 保存基础环境信息
* 保存地势信息

它不负责：

* 战斗逻辑
* 角色成长
* 技能效果
* 任务逻辑
* 剧情流程
* NPC 行为

### 2.3 数据驱动

地块本身保持轻量。

地图中的大部分玩法均由外部系统读取地块信息后完成，而不是由地块主动处理。

### 2.4 采用 ECS 式职责分离思想

整个地图系统应当采用 ECS（Entity-Component-System）式的职责拆分思想，即使最终代码实现不严格使用 ECS，也要按照 ECS 的边界来写设计文档。

这意味着：

* 地块（Tile）是世界中的一个坐标点
* NPC 通过 `currentTileId` 引用地块
* Boss 通过 `currentTileId` 引用地块
* 事件通过 `targetTileId` 引用地块
* 任务通过 `targetTileId` 引用地块
* 资源点通过 `tileId` 引用地块

不要写成：

* 地块拥有事件
* 地块拥有 NPC
* 地块拥有 Boss

而应当理解为：

所有对象共享同一套空间坐标系统，并通过引用地块完成定位。

这样以后新增大量玩法时，Tile 结构本身几乎不需要修改。

---

## 3. 地块结构

每一个单元地块都拥有唯一编号。

建议包含以下基础信息：

| 字段 | 说明 |
| --- | --- |
| 地块 ID | 唯一编号 |
| 坐标 | 地图中的位置（x, y, z） |
| 基础地形 | 当前地块所属自然地形 |
| 地势值 | 当前地块的相对地势 |
| 所属区域 | 当前区域编号 |
| 通行状态 | 是否允许进入 |

除此之外，不再保存其他复杂数据。

---

## 4. 地块职责

单元地块承担以下职责。

### 4.1 提供空间位置

所有角色、NPC、怪物、机关等对象都存在于某一个地块之上。

地图中的位置始终由地块决定。

### 4.2 提供邻接关系

每个正方形地块基于统一的 `x, y, z` 坐标系统组织相邻关系。

移动、攻击距离、范围判定均通过邻接关系计算。

在首版规则中，地图距离判断采用平方值比较原则：

* 计算 `dx^2 + dy^2`
* 比较其是否小于等于目标距离的平方

其中 `z` 不参与基础距离计算，而是用于表示地势并影响其他系统。

因此，系统不在每次判断时取平方根。

### 4.3 提供基础环境

地块拥有一种基础地形。

例如：

* 平原
* 森林
* 山地
* 沙漠
* 湖泊
* 河流

基础地形决定：

* 是否可通行
* 基础移动消耗
* 是否允许生成某些资源

具体规则由 [04-基础地形.md](../04-地图系统/04-基础地形.md) 定义。

### 4.4 提供地势信息

除基础地形外，地块还拥有一个独立的地势值（Elevation）。

地势值用于描述该地块相对于周边地块的高低差，并在坐标体系中对应 `z` 值。

地势会影响：

* 移动额外消耗
* 视野判断
* 战斗高低差修正

具体规则由 [04-基础地形.md](../04-地图系统/04-基础地形.md) 定义。

### 4.5 提供区域归属

每一个地块都属于一个区域。

例如：

* 百妖森林
* 赤焰王城
* 创界山
* 焚天荒漠

区域负责事件和环境。

地块仅保存所属区域编号。

### 4.6 提供通行状态

每个地块拥有一个基础通行状态。

例如：

* 可进入
* 不可进入
* 条件进入

具体条件由障碍系统决定。

---

## 5. 地块不负责的内容

为了保持系统稳定，以下内容均不属于单元地块职责。

### 5.1 玩家

玩家记录自己的位置。

而不是：

地块保存所有玩家。

### 5.2 NPC

NPC 拥有自己的位置。

地图查询当前位置即可。

### 5.3 怪物

怪物属于战斗系统。

不属于地块。

### 5.4 事件

事件属于事件系统。

事件发生在某个地块。

不是：

地块拥有事件。

### 5.5 任务

任务记录目标位置。

不是：

地块记录任务。

### 5.6 建筑

建筑属于地点系统。

地块只知道：

这里存在一个地点。

---

## 6. 地块与其他系统关系

```text
            玩家
              │
              │
角色系统 ───── 地块 ───── 地形系统
              │
              │
          区域系统
              │
              │
      事件 / 战斗 / 裂变
```

地块始终作为地图系统的基础容器。

所有系统共享同一套地块坐标。

---

## 7. 首版设计约束

`V1` 版本中，每个地块：

* 仅拥有一种基础地形
* 仅属于一个区域
* 不支持多层地图
* 不支持地下或空中高度
* 不支持多个区域重叠

所有扩展能力预留至后续版本。

---

## 8. 总结

单元地块是地图系统的最小空间单位。

它负责：

* 描述位置
* 描述邻接关系
* 描述基础环境
* 描述地势信息
* 描述区域归属
* 描述通行状态

除此之外，不承担任何复杂游戏逻辑。

地图中的角色、事件、任务、战斗、NPC、建筑和裂变等系统均通过引用地块完成空间定位，而不是耦合在地块对象内部。

这种设计能够保持地图系统长期稳定，并保证未来新增玩法时无需修改单元地块结构。
